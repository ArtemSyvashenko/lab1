## Опис проєкту

Цей проєкт автоматизує розгортання трикомпонентного вебзастосунку в ізольованій Docker-мережі за допомогою Docker Compose. Архітектура розроблена із забезпеченням суворої послідовності запуску та збереженням даних на диску хоста.

## Компоненти системи

* **nginx (0.0.0.0:80)** — єдина зовнішня точка входу. Перенаправляє трафік на бекенд.
* **webapp (127.0.0.1:3000)** — вебзастосунок на FastAPI. Закритий від зовнішнього світу.
* **db (MariaDB, 127.0.0.1:3306)** — СКБД для збереження завдань. Використовує персистентний том `db_data` (дані витримують видалення Docker чи перезапуск ОС).
* **migration-runner** — утилітарний контейнер для автоматичного створення таблиць у БД перед стартом основного застосунку.

## Послідовність запуску

`db` (чекає стан healthy) => `migration-runner` (виконує скрипт і завершується з кодом 0) => `webapp` (запускається та проходить healthcheck надійно через `urllib`) => `nginx` (піднімається останнім).
